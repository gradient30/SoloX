                    前置条件：SoloX 内置 ADB（platform-tools 34+）；电脑与手机同一 Wi-Fi；已开启 USB 调试并授权；无线调试需 ADB 30.0.0+（Android 11 起）。
    
                    2026 建议：Android 11–16 优先用方式二（无线调试配对）；Android 12+ 多数机型已禁用或未开放 adb tcpip，勿再依赖固定 5555 端口。
                
    

                
    方式一：传统 ADB TCP/IP（仅 Android 10 及以下，或已通过 USB 临时开启）
    
                
      
                    
    1. USB 连接电脑，确认 adb devices 显示 device
      2. 
                    
    2. 开启 TCP 监听（需 root 或已授权调试）：
      adb tcpip 5555
      3. 
                    
    3. 查看手机 IP：设置 → WLAN → 当前网络详情，或：
      adb shell ip -4 addr show wlan0
      4. 
                    
    4. 拔掉 USB，无线连接：
      adb connect <手机IP>:5555
      5. 
                    
    5. 验证：adb devices 应出现 <IP>:5555 device
      6. 
                
    
                
    
                    限制：Android 11+ 非 root 设备上 adb tcpip 常返回错误或被 OEM 屏蔽；重启手机后需重新执行。Android 12+ 请改用方式二。
                
    

                
    
                
    方式二：无线调试配对（Android 11–16，官方推荐）
    
                
      
                    
    1. 手机进入 设置 → 开发者选项 → 无线调试，开启开关（Android 13+ 可能需同时开启「USB 调试」）
      2. 
                    
    2. 点击 使用配对码配对设备，记录 6 位配对码 与 IP:配对端口（如 192.168.1.8:37123）
      3. 
                    
    3. 电脑配对（端口为配对端口，非调试端口）：
      
                        adb pair 192.168.1.8:37123
      
                        提示时输入配对码；或使用 Android Studio / SDK 配对对话框
      4. 
                    
    4. 返回无线调试主界面，查看 IP 地址和端口（调试端口，每次开关无线调试可能变化，如 192.168.1.8:41567）
      5. 
                    
    5. 连接：
      adb connect 192.168.1.8:41567
      6. 
                    
    6. 验证：adb devices；刷新 SoloX 主页设备列表
      7. 
                
    
                
    
                    Android 14/15/16 提示：配对码约 60 秒过期；企业策略可能禁用无线调试；首次 USB 连接仍需点「允许 USB 调试」。
                
    

                
    
                
    方式三：mDNS 自动发现（Android 12+，ADB 31+，可选）
    
                
      
                    
    1. 完成方式二配对后，在电脑执行：
      adb mdns services
      2. 
                    
    2. 找到 _adb._tcp 条目，按提示连接：
      adb connect <设备名>._adb._tcp.local:<端口>
      或直接 adb connect <IP>:<调试端口>
      3. 
                
    

                
    
                
    断开连接
    
                
    adb disconnect <IP>:<端口>    # 断开指定设备
adb disconnect                   # 断开所有无线设备
adb kill-server && adb start-server   # 重置 ADB 服务

    问题解决方法
    
                    
                        
    error: protocol fault / 配对失败升级 platform-tools；确认配对码未过期；关闭 VPN/代理；Windows 防火墙放行 adb.exe
    
                        
    unable to connect同一 Wi-Fi（勿用访客网络隔离）；ping 手机 IP；核对调试端口非配对端口
    
                        
    cannot connect to daemonadb kill-server && adb start-server 后重试
    
                        
    device unauthorizedUSB 重新授权；无线调试界面点「撤销授权」后重配
    
                        
    连接频繁断开加强 Wi-Fi 信号；关闭省电/休眠断 Wi-Fi；勿与投屏高码率同时占满带宽
    
                        
    SoloX 看不到设备终端先确认 adb devices 正常，再刷新浏览器设备下拉框

                    Prerequisites: Bundled ADB (platform-tools 34+); phone and PC on same Wi-Fi; USB debugging authorized; wireless debugging needs ADB 30.0.0+ (Android 11+).
    
                    2026 recommendation: Use Method 2 (wireless debugging) on Android 11–16; do not rely on fixed port 5555 on Android 12+.
                
    

                
    Method 1: Legacy ADB TCP/IP (Android 10 and below, or USB-initiated)
    
                
      
                    
    1. Connect via USB; verify adb devices shows device
      2. 
                    
    2. Enable TCP listener:
      adb tcpip 5555
      3. 
                    
    3. Find IP: Settings → Wi-Fi → network details, or:
      adb shell ip -4 addr show wlan0
      4. 
                    
    4. Unplug USB and connect:
      adb connect <PHONE_IP>:5555
      5. 
                    
    5. Verify: adb devices shows <IP>:5555 device
      6. 
                
    
                
    
                    Limit: adb tcpip is blocked or restricted on most Android 12+ devices without root. Use Method 2 instead.
                
    

                
    
                
    Method 2: Wireless debugging pairing (Android 11–16, recommended)
    
                
      
                    
    1. Enable Settings → Developer options → Wireless debugging
      2. 
                    
    2. Tap Pair device with pairing code; note the code and IP:pairing_port
      3. 
                    
    3. On PC:
      adb pair <IP>:<PAIRING_PORT>
      Enter the pairing code when prompted
      4. 
                    
    4. On the Wireless debugging screen, note IP address and port (debug port, changes when toggling)
      5. 
                    
    5. Connect:
      adb connect <IP>:<DEBUG_PORT>
      6. 
                    
    6. Verify with adb devices; refresh SoloX device list
      7. 
                
    
                
    
                    Android 14–16: Pairing codes expire in ~60s; MDM may disable wireless debugging.
                
    

                
    
                
    Method 3: mDNS discovery (Android 12+, ADB 31+, optional)
    
                
      
                    
    1. After pairing: adb mdns services
      2. 
                    
    2. Connect using the listed _adb._tcp service or adb connect IP:debug_port
      3.

    ProblemSolution
