性能指标采集说明

指标 采集来源 采集方法 正常范围 已知问题
CPU dumpsys cpuinfo / top 按核心采集进程CPU占用率，总值=各核心之和/核心数 平均 <60%，单核 <90% 多核求和可能超过100%
Memory (PSS) dumpsys meminfo <pkg> PSS = 私有内存 + 按比例分摊的共享内存 因应用类型而异 GC期间PSS会波动
FPS SurfaceFlinger --latency / gfxinfo framestats 帧时间戳计算：(帧数-1) / 时间跨度 稳定30或60帧 游戏引擎必须使用SurfaceView模式；桥接时间戳已移除
Jank 帧时间戳间隔 PerfDog标准：>2倍vsync间隔 且 >2倍最近3帧均值 Jank <2%，Big Jank <0.5% 旧阈值(166ms)偏高，现使用基于vsync的计算
GPU dumpsys gfxinfo GPU渲染时间 每帧 <16ms (60fps) 并非所有设备都支持
Network /proc/net/xt_qtaguid/stats 基于UID的发送/接收字节数差值 因应用而异 某些设备可能需要Root权限
Battery dumpsys battery 电量百分比、温度、电压 温度 <45°C 充电状态会影响读数
Disk dumpsys diskstats 读写统计 - -
Thermal dumpsys thermalservice CPU/GPU/电池温度传感器 CPU <50°C 传感器名称因厂商而异

补充说明

Android vs iOS 差异
  • Android 通过 ADB 命令采集（dumpsys、top、/proc 文件系统等）
  • iOS 通过 tidevice / instruments 协议采集，部分指标采集方式不同
SurfaceView vs 标准View检测
  • SoloX 自动检测游戏引擎（Unity/UE4/UE5/Cocos/Laya），并强制使用 SurfaceView 模式采集FPS
  • SurfaceView模式：使用 SurfaceFlinger --latency 获取帧数据
  • 标准View模式：使用 gfxinfo framestats（仅适用于View体系的应用，不适用于游戏引擎）
  • 回退链：多Surface重试 → 页面翻转计数 → gfxinfo总帧数
FPS 可信度元数据
  • 每次采集返回可信度信息：数据源、新帧数量、是否已验证、可信度等级
  • 可信度等级：high（直接帧时间戳）、medium（页面翻转计数）、low（gfxinfo总帧数）、none（无数据）

Additional Notes

Android vs iOS Differences
  • Android collects via ADB commands (dumpsys, top, /proc filesystem, etc.)
  • iOS collects via tidevice / instruments protocol; some metrics differ in collection method
SurfaceView vs Standard View Detection
  • SoloX auto-detects game engines (Unity/UE4/UE5/Cocos/Laya) and forces SurfaceView mode for FPS
  • SurfaceView mode: Uses SurfaceFlinger --latency for frame data
  • Standard View mode: Uses gfxinfo framestats (only works for View-based apps, not game engines)
  • Fallback chain: multi-surface retry → page flip count → gfxinfo total frames
FPS Confidence Metadata
  • Each collection returns confidence info: source, fresh_frame_count, verified, confidence level
  • Levels: high (direct frame timestamps), medium (page flip count), low (gfxinfo total frames), none (no data)
