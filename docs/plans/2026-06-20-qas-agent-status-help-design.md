# QAS Network Agent 状态闭环与帮助页设计

## 背景

当前 QAS Network Agent 已具备中文 4 Tab 原生界面，但总览页按钮点击后的状态反馈不够明确，容易让测试人员无法判断“这是介绍文案”还是“真实运行状态”。日志筛选缺少视觉层级，帮助信息也不足以让非计算机专业测试人员理解弱网链路。

## 目标

- 建立 Activity 与 VpnService 之间的轻量状态闭环，减少“按钮点了没变化”的不确定性。
- 总览页明确区分真实检测状态、最近操作结果和说明文案。
- 操作按钮点击后立即反馈，并可通过刷新读取当前状态。
- 日志按级别显示颜色区分，正文更轻，重点更突出。
- 整体内容区域使用不同颜色强化视觉层级。
- 增加 `?` 帮助弹层，用非专业语言解释弱网实现原理，并展示链路图。
- 修正 VPN 状态栏图标误解：系统 VPN 图标只在真实 VPN 隧道建立后出现，单纯授权或前台服务启动不会显示。

## 状态真实性边界

可真实检测：

- VPN 授权：`VpnService.prepare(activity) == null`。
- Agent 前台服务生命周期：由 `SoloXVpnService.onCreate/onDestroy` 写入共享状态。
- VPN 隧道状态：由 `startTunnel/stopTunnel/onDestroy` 写入共享状态。
- 最近操作：Activity 按钮点击后写入共享状态和本地日志。

不可伪造：

- Android 顶部状态栏 VPN 图标。该图标由系统控制，只有 `VpnService.Builder.establish()` 成功建立真实 VPN 网络后才会出现。
- 只点击“授权并启动后台服务”时，如果 SoloX 控制端尚未下发目标 App 和弱网 profile，不会建立 VPN 隧道，也不应期待系统 VPN 图标出现。

## 信息架构

### 顶部标题栏

- 左侧：`QAS 弱网代理`
- 右侧：`?` 帮助按钮
- 帮助按钮打开对话框/弹层，不离开当前页。

### 总览

状态卡片展示：

- 授权状态：已授权 / 待授权。
- 服务状态：运行中 / 已停止。
- 隧道状态：已建立 / 未建立 / 错误。
- 最近操作：例如“已请求启动后台服务”、“已停止服务”、“状态已刷新”。
- VPN 图标提示：只有隧道建立后系统状态栏才会显示 VPN 图标。

快捷操作：

- `▶ 启动`：请求 VPN 授权或启动后台服务。
- `■ 停止`：停止后台服务并更新状态。
- `⟳ 刷新`：重新读取共享状态与授权状态。
- `⚙ VPN`：打开 Android VPN 设置页或系统设置页。

按钮尽量一行显示；窄屏允许换行，不牺牲可读性。

### 弱网

继续说明：

- 目标包名由 SoloX 控制端下发。
- 只捕获目标 App UID。
- 数据链路为 `TUN -> tun2proxy -> SOCKS5 shaper`。

增加视觉层级：

- 数据面说明用蓝色提示卡。
- 安全边界用橙色提示卡。

### 日志

- 筛选使用轻量 chip/分段按钮：全部、错误、警告、信息、调试。
- 日志条目使用彩色条或背景：
  - `ERROR` 红色。
  - `WARN` 橙色。
  - `INFO` 蓝/绿。
  - `DEBUG` 灰色。
- 序号、级别、来源突出；消息正文缩小。

### 设置

保留版本、包名、控制 socket、后台运行方式。

## 帮助弹层内容

面向非计算机专业测试人员，用简单语言解释：

1. SoloX 在电脑端选择目标 App 和弱网模板。
2. 手机端 QAS Agent 请求 Android VPN 权限。
3. 真实测试开始时，Agent 只接管目标 App 的网络，不影响其他 App。
4. 网络先进入 Agent 的虚拟网卡 TUN。
5. tun2proxy 把网络流量转成代理请求。
6. SOCKS5 shaper 按延迟、丢包、带宽规则处理后再访问真实网络。

链路图：

```text
目标 App
  ↓
Android VPN（只接管目标 UID）
  ↓
QAS Agent / TUN
  ↓
tun2proxy
  ↓
SOCKS5 弱网整形
  ↓
真实网络
```

帮助页同时说明：

- 为什么需要 VPN 授权。
- 为什么授权后不一定马上出现 VPN 图标。
- 为什么要点击清理并确认没有 VPN 残留。

## 实现策略

- 保持原生 Java、无外部 UI 依赖。
- 新增 `AgentUiState` 或等价类，封装 `SharedPreferences` 状态读写。
- `SoloXVpnService` 在生命周期和隧道 start/stop 处写状态。
- `MainActivity` 在按钮点击和刷新时读取状态并重绘总览页。
- UI 使用 `TextView`/`Button`/`LinearLayout`/`GradientDrawable` 实现卡片、状态条、chip。
- 使用 `AlertDialog` 显示帮助弹层。

## 验收标准

- 源码测试证明 UI 包含真实状态字段、帮助弹层、VPN 图标边界说明、系统 VPN 设置跳转。
- 源码测试证明服务生命周期和隧道状态会写入共享状态。
- 日志页源码包含各级别颜色映射。
- Android Agent 相关测试通过。
- Android release 构建成功，公开 APK 更新。
- 真机安装启动后：
  - 总览页展示授权/服务/隧道/最近操作。
  - 启动、停止、刷新按钮点击后状态文字变化。
  - 帮助弹层可打开，含链路图。
  - 日志页不同级别具备颜色区分。
  - 未建立真实隧道时，不把“无 VPN 图标”误报为失败。
